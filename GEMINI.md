\# Project: Interview-Auto (실전형 AI 다중 에이전트 면접 프로그램)



\## 1. 프로젝트 개요 (Overview)

본 프로젝트는 사용자가 지원하는 '기업 정보'와 '자기소개서/이력서'를 바탕으로, 실제 면접과 유사한 환경을 제공하는 대화형 모의 면접 프로그램입니다. 

전작인 `CV-Auto`의 아키텍처(Streamlit UI, 로컬 캐시 시스템, Google Search 통합)를 계승하며, \*\*OmG (Context-engineering-powered multi-agent workflow)\*\*를 도입하여 3인의 면접관과 1인의 피드백 분석가 등 다중 에이전트 환경을 구축합니다.



\## 2. 개발 환경 및 기술 스택 (Tech Stack)

\* \*\*Language \& UI Framework\*\*: Python, Streamlit (전작과 동일한 UI 테마/구조 유지)

\* \*\*LLM Core\*\*: 사용자 지정 모델 (메인 대화 및 피드백 처리용), Gemma 계열 (필요 시 초경량 분류용)

\* \*\*Multi-Agent Framework\*\*: OmG (Gemini CLI Extension) - 다중 면접관 상태 유지 및 워크플로우 제어

\* \*\*File Parsing\*\*: `pdfplumber` 또는 `PyPDF2` (PDF 지원), 기본 파일 I/O (MD, TXT)

\* \*\*Search/Cache\*\*: Google Search Tool 통합 및 로컬 디렉토리 기반 캐싱 (`search\_utils.py`, `cache\_manager.py` 활용)



\## 3. 핵심 워크플로우 (Core Workflow)



\### Step 1: 초기 설정 및 정보 수집 (Initialization)

1\. \*\*타겟 정보 입력\*\*: 지원 기업, 직무를 입력하면 Google Search Tool을 통해 최신 기업 정보(인재상, 핵심 가치, 최신 뉴스)를 검색 및 캐싱합니다.

2\. \*\*서류 업로드\*\*: 사용자가 이력서 및 자기소개서(.md, .txt, .pdf)를 Drag \& Drop으로 업로드합니다.

3\. \*\*사전 분석\*\*: OmG 에이전트들이 업로드된 서류와 기업 정보를 바탕으로 면접 질문 풀(Pool)을 사전 생성합니다.



\### Step 2: 실전 모의 면접 (Interactive Interview)

1\. 3인의 면접관 에이전트가 번갈아 가며 질문을 던집니다.

2\. 사용자가 답변을 입력하면, 이전 답변의 논리적 허점을 파고드는 \*\*꼬리 질문(Follow-up)\*\*이 동적으로 생성됩니다.



\### Step 3: 실시간 피드백 (Real-time Feedback)

1\. 사용자가 답변을 제출할 때마다 백그라운드에서 '피드백 에이전트'가 답변을 평가합니다.

2\. 두괄식 전개, 직무 적합성, 논리력 등을 평가하여 UI의 우측 화면에 즉각적으로 교정 내용과 모범 답변 방향을 제시합니다.



\## 4. UI/UX 레이아웃 구조 (Streamlit Layout)

화면은 `st.columns(\[1.2, 1])` 비율로 분할합니다.



\* \*\*\[Sidebar]\*\*: API 키 설정, 진행 상태 초기화, 세션 관리 기능.

\* \*\*\[Column Left] 대화 및 입력 영역 (Chat Window)\*\*:

&#x20;   \* 상단: 파일 업로드 컴포넌트 (.md, .txt, .pdf 지원).

&#x20;   \* 중단: `st.chat\_message`를 활용한 면접관(Agent) 질문과 사용자(User) 답변의 핑퐁 대화 내역.

&#x20;   \* 하단: 사용자 답변 입력용 `st.chat\_input` 또는 `st.text\_area`.

\* \*\*\[Column Right] 실시간 분석 및 교정 (Live Feedback)\*\*:

&#x20;   \* 답변이 등록될 때마다 실시간으로 업데이트되는 평가 패널.

&#x20;   \* \*\*체크 항목\*\*: 답변의 명확성, 기술적/경험적 근거의 충분성, 면접관 의도 파악 여부.

&#x20;   \* \*\*교정 가이드\*\*: "이런 식으로 답변을 수정해 보세요" 형태의 실시간 가이드 제공.



\## 5. OmG Multi-Agent 페르소나 및 역할 정의 (Agent Personas)



OmG 워크플로우를 활용하여 다음 4개의 에이전트를 구성합니다. 모든 에이전트는 사용자의 서류와 기업 분석 정보를 공유(Shared Context)합니다.



1\. \*\*실무진 면접관 에이전트 (Agent\_Tech)\*\*

&#x20;  \* \*\*역할\*\*: 직무 역량, 프로젝트 디테일, 전공 지식 검증.

&#x20;  \* \*\*성향\*\*: 날카롭고 꼬리 질문을 선호함. "그 기술을 선택한 이유는 무엇인가요?", "해당 프로젝트에서 발생한 트러블슈팅 과정을 수치화해서 설명해 주세요" 등 구체적인 답변을 요구함.

2\. \*\*인사담당자 에이전트 (Agent\_HR)\*\*

&#x20;  \* \*\*역할\*\*: 조직 적합성(Fit), 인성, 커뮤니케이션 능력, 갈등 해결 경험 확인.

&#x20;  \* \*\*성향\*\*: 부드럽지만 핵심을 찌름. "팀원과 의견 충돌이 있었을 때 어떻게 해결했나요?" 등의 행동 기반 면접(BEI) 질문 담당.

3\. \*\*임원진 면접관 에이전트 (Agent\_Exec)\*\*

&#x20;  \* \*\*역할\*\*: 거시적 관점, 압박 질문, 로열티 및 가치관 확인.

&#x20;  \* \*\*성향\*\*: 단호하고 묵직함. "우리 회사가 왜 지원자를 뽑아야 합니까?", "본인의 단점이 치명적으로 작용할 수 있는데 어떻게 생각하나요?" 등 스트레스 상황 대처를 확인.

4\. \*\*AI 피드백 컨설턴트 (Agent\_Analyst)\*\*

&#x20;  \* \*\*역할\*\*: 메인 UI 우측에 출력되는 실시간 교정 전담.

&#x20;  \* \*\*성향\*\*: 객관적이고 건설적. 사용자의 대화 흐름에 직접 개입하지 않고 뒷단에서 답변을 분석해 JSON 형태로 피드백 데이터를 Streamlit UI로 전달함.



\## 6. 개발 및 코드 작성 지침 (Development Guidelines)

1\. \*\*코드 재사용성\*\*: 첨부된 `CV-Auto`의 `search\_utils.py`, `cache\_manager.py`, `logger.py`를 최대한 재활용하여 모듈러(Modular) 구조를 유지할 것.

2\. \*\*비동기/상태 관리\*\*: Streamlit의 `st.session\_state`를 활용하여 OmG의 에이전트 대화 히스토리와 피드백 상태가 초기화되지 않도록 철저히 관리할 것.

3\. \*\*코드 리뷰\*\*: `gemini 3.1 pro preview`를 통해 각 모듈(파일 파서, 에이전트 통신, UI 렌더링) 작성 후 반드시 사이드 이펙트와 프롬프트 주입(Prompt Injection) 방어에 대한 코드 리뷰를 진행할 것.
